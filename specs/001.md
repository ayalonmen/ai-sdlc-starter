## Types

```typescript
type MemberId = string;

interface Member {
  id: MemberId;
  name: string;
}

interface Expense {
  id: string;
  payerId: MemberId;
  amount: number;
  sharerIds: MemberId[];
}

interface Group {
  id: string;
  members: Member[];
  expenses: Expense[];
}

type ValidationErrorCode = 'EMPTY_NAME' | 'DUPLICATE_NAME' | 'MEMBER_HAS_EXPENSES';

interface ValidationError {
  code: ValidationErrorCode;
  message: string;
}

type AddMemberResult =
  | { ok: true; group: Group; member: Member }
  | { ok: false; error: ValidationError };

type RemoveMemberResult =
  | { ok: true; group: Group }
  | { ok: false; error: ValidationError };
```

## Pure functions

- `normalizeMemberName(rawName: string): string` ‚Äî trims leading/trailing whitespace; used as the canonical form for emptiness and duplicate checks.
- `isNameEmpty(normalizedName: string): boolean` ‚Äî true if the normalized name has zero length.
- `isDuplicateName(normalizedName: string, members: Member[]): boolean` ‚Äî true if any existing member's normalized, lower-cased name equals `normalizedName.toLowerCase()`.
- `isMemberReferencedInExpenses(memberId: MemberId, expenses: Expense[]): boolean` ‚Äî true if `memberId` equals any expense's `payerId` or appears in any expense's `sharerIds`.
- `addMember(group: Group, rawName: string): AddMemberResult` ‚Äî normalizes `rawName`; returns `{ ok: false, error }` with code `EMPTY_NAME` or `DUPLICATE_NAME` if validation fails; otherwise returns `{ ok: true, group: <new group with member appended>, member }` without mutating the input `group`.
- `removeMember(group: Group, memberId: MemberId): RemoveMemberResult` ‚Äî returns `{ ok: false, error }` with code `MEMBER_HAS_EXPENSES` if `isMemberReferencedInExpenses` is true; otherwise returns `{ ok: true, group: <new group with member filtered out> }` without mutating the input `group`.

## Engine

```typescript
type GroupAction =
  | { type: 'ADD_MEMBER'; rawName: string }
  | { type: 'REMOVE_MEMBER'; memberId: MemberId };

function applyGroupAction(group: Group, action: GroupAction): AddMemberResult | RemoveMemberResult;
```

- Dispatches to `addMember` or `removeMember` based on `action.type`.
- Never mutates the `group` passed in; on `ok: true` the caller replaces the in-memory group with the returned `group` and triggers the existing localStorage flush; on `ok: false` the caller displays `error.message` inline and leaves in-memory state and localStorage untouched.

## Rules encoded

- `EMPTY_NAME_MESSAGE`, `DUPLICATE_NAME_MESSAGE`, `MEMBER_HAS_EXPENSES_MESSAGE` ‚Äî fixed, user-facing copy returned in `ValidationError.message` for each respective code.
- **Required name rule**: a name is valid only if `normalizeMemberName(rawName)` is non-empty; whitespace-only input is treated as empty.
- **Uniqueness rule**: uniqueness is case-insensitive and scoped to the single `Group.members` array; comparison is performed on trimmed names.
- **Removal-block rule**: a member may be removed only if `isMemberReferencedInExpenses` returns false for both `payerId` and every entry in `sharerIds` across all of the group's expenses.
- **Atomicity rule**: `addMember` and `removeMember` return a new `Group` object only on success; on failure they return the error variant and perform no mutation ‚Äî the caller must not write a failed result back into state or localStorage.
- **Persistence rule**: after any `ok: true` result from `applyGroupAction`, the returned `group` (including `members` and pre-existing balance data) is flushed to localStorage via the existing persistence layer, and rehydration on load must reconstruct an equivalent `Group` object.

## Criteria to spec mapping

| Criterion | Spec element |
|---|---|
| 1 | `addMember` success path ‚Üí `{ ok: true, group, member }` |
| 2 | `isNameEmpty` + `EMPTY_NAME` error in `addMember` |
| 3 | `isDuplicateName` + `DUPLICATE_NAME` error in `addMember` |
| 4 | `removeMember` success path when `isMemberReferencedInExpenses` is false |
| 5 | `isMemberReferencedInExpenses` (payer case) + `MEMBER_HAS_EXPENSES` error in `removeMember` |
| 6 | `isMemberReferencedInExpenses` (sharer case) + `MEMBER_HAS_EXPENSES` error in `removeMember` |
| 7 | Atomicity rule: `addMember`/`removeMember` return new objects only on `ok: true`, never mutate input `group` |
| 8 | Persistence rule under Rules encoded |
